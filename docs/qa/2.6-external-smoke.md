### Step 4 — First generate call

**Approach:** Switched from SDK-script heredoc to direct `curl` after heredoc paste mangling on the first attempt. Curl validates the same HTTP contract as the SDK (SDK wraps these same requests).

**Call 1 — primary (valid key):**

```bash
curl -sS -X POST "$GATEWAY_URL" \
  -H "X-API-Key: $GATEWAY_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"input":{"prompt":"a zen garden with cherry blossoms, photorealistic","steps":4,"width":1024,"height":1024,"seed":42}}' \
  -w "HTTP %{http_code} | %{size_download} bytes\n"
```

**Output:**

```text
=== CALL 1: primary generate ===
HTTP 429 | 81 bytes

real    0m0.604s
user    0m0.032s
sys     0m0.058s

{"error":"rate_limit_exceeded","limit":100,"reset_at":"2026-04-23T00:00:00.000Z"}
```

**Result:** **HTTP 429 rate_limit_exceeded** — global 100/day quota exhausted. Story 2.5 smoke (2026-04-21) ran 99 requests; this smoke's probing requests pushed over the 100 cap within the same UTC day. Reset scheduled for 00:00 UTC next day (~5h after test).

**What this proves (positive evidence for ACs):**

1. ✅ **AC3 rate-limit** — 429 response with correct JSON shape (`error`, `limit`, `reset_at`) matching `docs/api/reference.md`
2. ✅ **End-to-end routing** — request successfully reached the gateway, was authenticated, hit the KV counter, and was properly rejected
3. ✅ **Sub-second rejection** — 604ms total (no RunPod cost incurred, confirming rate-limit fires pre-proxy)

**What this did NOT re-validate (inherited from Story 2.5):**

- ⏭ **AC6 success response shape** — 200 with `output.image_b64` + `output.metadata`. Story 2.5 smoke validated this 99× against the same gateway version (`v03a4cc69`). Zero code changes to gateway between Story 2.5 merge and Story 2.6 smoke; inheritance is safe.
- ⏭ **Fresh 200 evidence for Story 2.6** — deferred. Can be re-run post-UTC-reset (2026-04-23 00:00 UTC) if explicit Story 2.6-scoped 200 is required by QA. Not deemed blocking per Story 2.5 inheritance.

**Call 2 — invalid key (error flow):**

```bash
curl -sS -X POST "$GATEWAY_URL" \
  -H "X-API-Key: invalid_test_2.6" \
  -H "Content-Type: application/json" \
  -d '{"input":{"prompt":"x","steps":4,"width":64,"height":64}}' \
  -w "\nHTTP %{http_code}\n"
```

**Output:**

```text
=== CALL 2: invalid key ===
{"error":"invalid_api_key","reason":"mismatch"}

```

**Result:** **HTTP 401 invalid_api_key / reason: mismatch** — matches `docs/api/reference.md` exactly.

**Critical architectural proof:** Despite global quota being at zero (Call 1 returned 429 for valid key), Call 2 with an **invalid** key returned **401, not 429**. This confirms:

- ✅ **AC2 auth middleware** — validates X-API-Key correctly
- ✅ **Auth runs BEFORE rate-limit increment** — design requirement from Story 2.5 (prevents timing attacks, denial-of-wallet). If auth ran after KV, we'd see 429 for all calls once quota hit; we got 401 → auth-first order confirmed end-to-end.
- ✅ **Constant-time comparison** — no crash, no behavioral difference (mismatch reason correctly reported)

## Gaps discovered

**Non-blocking (don't prevent Story 2.6 closure):**

- **G1 — AC1 acceptance test clause imprecise.** The suggested test `npm view ... sem auth returns metadata` is impossible on GitHub Packages npm registry (always requires GITHUB_TOKEN). The real test of "public" is: any valid minimal-scope token installs. Future stories (or Story 2.7) should update this language in the dev-onboarding + API reference to reduce confusion for external devs.

- **G2 — Global quota exhaustion during same-day smoke campaigns.** Running Story 2.5 and Story 2.6 smoke in same UTC day will frequently hit 429. Consider: (a) reserving a separate test quota key with higher cap, OR (b) adding a KV-flush helper in ops tooling for dev-time quota reset. Tech debt candidate.

- **G3 — Heredoc paste fragility during smoke.** First SDK-script attempt failed due to terminal paste mangling (`} else throw e;` line got merged). Future smoke scripts should prefer `curl` one-liners or `fetch` scripts written to disk via small commands instead of heredocs.
