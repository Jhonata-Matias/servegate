- 3× "Skip to content" anchors at strategic points

This passes the WCAG AA bar from a structural/semantic perspective. AAA would require additional manual review for elements like motion reduce, focus visible refinements, etc. — V2 scope.

## What I CANNOT verify from CLI (subjective + visual)

These are the things you (or `@devops` after Vercel preview is up) need to eyeball:

1. **Font rendering quality at retina / non-retina** — Inter and JetBrains Mono fall back to system if not bundled. On macOS this is invisible (Inter is system-installed via SF Pro fallback chain). On Windows / Linux Chrome, it may fall back to default sans-serif, which looks worse than landing. **Fix if needed:** bundle Inter + JetBrains Mono via `@fontsource/inter` + `@fontsource/jetbrains-mono` (~150KB total, V2 polish).

2. **Spacing harmony** — Starlight's default vertical rhythm is professional but might not match the landing's deliberately roomier hero. The Welcome page has a different feel from the landing's hero — that's expected (we replaced marketing with docs), but verify it doesn't feel cramped.

3. **Card hover states** — my `theme.css` includes a hover border-color shift to accent. Verify it animates smoothly (not jarring) and that all 6 cards respond consistently.

4. **Mobile responsiveness at 320px / 375px / 768px** — Starlight's default responsive design is solid, but the landing was hand-tuned. Check:
   - Cards stack to 1-col below 768px without wrapping ugly
   - Sidebar collapses to a hamburger drawer on mobile
   - TOC right-side hides on tablet+ (it should)
   - Touch targets ≥ 44×44 px on tablet/mobile
   - No horizontal scroll on 320px

5. **Pagefind search box placement** — I confirmed it exists, but didn't verify visual prominence. Should be top-of-sidebar and trigger via `Ctrl/Cmd+K` (Starlight default).

6. **Code block styling inside MDX** — code examples use Shiki syntax highlighting. Verify the colors don't clash with the dark theme. Also check that long code lines wrap or scroll horizontally cleanly.

7. **Link colors inside Asides** — the AC8 fix changed Aside backgrounds to dark, but link colors inside Asides might now be too saturated against the dark surface. Spot-check the 4 pages with Asides (quickstart/api/sdk/errors).

8. **Sidebar item visual weight** — group label "Get started" should be visually distinct from the group items below it.
