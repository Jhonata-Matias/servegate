## Deviation from Story 2.6 precedent (non-blocking)

**Story 2.6 AC7 used Docker `node:20` container for "maximally clean environment" validation.** This smoke used **host Node v24.14.0 in a fresh tmpdir outside the repo** because the Docker daemon socket on this host required group membership not available to the current user. Assessment:

| Criterion | Docker (2.6 precedent) | Host fresh tmpdir (2.8-v0.2.0) | Delta impact |
|---|---|---|---|
| Clean `node_modules` (no parent pollution) | ✅ guaranteed by container | ✅ guaranteed by tmpdir outside repo | None |
| Clean `.npmrc` (no parent auth inheritance) | ✅ dedicated file in container | ✅ dedicated file in tmpdir | None |
| Clean `tsconfig` / `package.json` parent chain | ✅ `/work` isolated | ✅ `/tmp/flux-smoke-v020-*` isolated | None |
| Node version parity with docs | v20 LTS (docs: >=18) | v24 current (docs: >=18) | None (both satisfy engines constraint) |
| Proves "external dev" journey | Yes — Docker = generic "outside" | Yes — fresh tmpdir + dedicated auth = generic "outside" | Equivalent confidence |

**Conclusion:** The "external dev" journey claim (Story 2.6 AC7 pattern) holds. A follow-up in Docker can be deferred to the next alpha-refresh smoke if desired; not blocking v0.2.0 release.

## Non-findings (things that could have gone wrong, didn't)

- **ColdStartError stale import:** sample code in `sdk/README.md`, `dev-onboarding.md` (EN+pt-BR) was updated in commit `1af3768` before publish. Zero drift between docs and published artifact.
- **Rate-limit collision:** 2 images consumed (warmup + 1 generate) — well within daily quota.
- **Cold start:** worker happened to be warm (recent INC deploy traffic). Cold start scenario is covered by existing gateway `integration.test.ts` in gateway phase 3 suite (passing 65/65 before publish).
- **Bundle leak:** smoke proves `apiKey` is only in the `FluxClient` constructor; never serialized into `.js` files. Secret handling validated indirectly (SDK test suite `bundle-cjs`/`bundle-esm` covers grep-level assertions; not rerun here since no SDK source change since those tests passed).
