### F2 — pt-BR currency variant `/mês` missed by sed rules (MEDIUM)

**Location:** Multiple sanitized files — pt-BR cost terminology `$X/mês` (portuguese for "per month") not covered by Rule 2 sed patterns which only matched `/mo` (English).

**Affected files + counts:**
| File | Matches |
|---|---|
| `docs/stories/2.1.2.decide-cold-start-mitigation.story.md` | 1 (`$20/mês 24/7 idle`) |
| `docs/stories/2.1.runpod-serverless-flux-endpoint.story.md` | 3 (`$16/mês`, `$500/mês`, `~$50/mês`) |
| `docs/stories/2.5.gateway-rate-limit-cloudflare.story.md` | 1 (`$1.80/mês máx`) |
| `docs/architecture/adr-0001-flux-cold-start.md` | 3 (`$544/mês`, `$337/mês`, `$180/mês`) |
| `docs/qa/1.1-closure-summary.md` | 2 (`$5/mês`, `$2.50/mês`) |
| `docs/prd/epic-2-consumer-integration.md` (OUTSIDE methodology section) | 4 (`$9/mês`, `$8/mês`, `$4.50/mês`, `$360/mês`) |

**Total:** **14 pt-BR cost variants** remain after Story 2.8 sanitization.

**Root cause:** Security audit Section 7 Rule 2 examples used `/mo` form (e.g., `$25/mo budget`); @dev's sed script replicated these patterns literally. pt-BR source files use `/mês` suffix for monthly costs, a variant not covered.

**Recommendation:** Extend sed rules to cover `/mês` variants:
```sed
s/\$[0-9]+\.[0-9]+\/mês/<cost per month>/g
s/\$[0-9]+\/mês/<cost per month>/g
```
OR more conservative per-match substitution (same mapping as English counterparts with explicit pt-BR handling). Single re-run on affected files.

**Fix effort:** 10-15 minutes (extend sed script + re-run on 6 files + re-verify).

**Root cause:** Sed rule ordering — `$25/mo → <$50/mo alpha budget` ran first, then `$50/mo → <cost threshold>` ran again on the intermediate result.

**Severity:** LOW — functionally correct (no literal cost exposed), just visually awkward.
