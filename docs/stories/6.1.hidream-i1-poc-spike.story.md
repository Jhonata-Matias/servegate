## User Story

**As** a servegate maintainer evaluating whether HiDream-I1 Dev is operationally viable as a higher-fidelity open-source T2I successor to FLUX schnell,
**I want** an isolated RunPod L40S serverless endpoint that runs HiDream-I1 Dev FP16 with empirical measurements of cold start (P50/P95), warm inference latency (P50/P95), and per-image cost across a 50-prompt standardized set,
**so that** Story 6.2 can A/B-test these outputs against FLUX dev references and ADR-0006 can issue an adopt/reject/defer verdict backed by numbers — not speculation — with zero risk to the production FLUX T2I path serving live alpha customers.

---

## Context

### Why this story exists now

The 2026-05-06 brainstorming session (Atlas analyst, multi-technique) concluded that **HiDream-I1 Dev (17B MoE Apache-2.0, ~6B active)** is the highest-promise candidate for a higher-fidelity open-source T2I path that fits servegate's existing operational pattern (RunPod serverless + R2 + Cloudflare Workers gateway). Knowledge-cutoff caveats flagged HiDream-specific details (cold start, $/imagem, real-world quality vs FLUX dev) as **⚠️ requires empirical verification**. Epic 6 charters the verification.

The asymmetry is sharp: committing infrastructure-as-code work (Epic 7+) on HiDream without empirical validation is high-cost on rework if the model turns out unfit. A bounded ~$50 PoC de-risks Epic 7 entirely.

This story is the **infrastructure measurement leg** of Epic 6. Story 6.2 (parallel) defines the evaluation prompt set, generates FLUX dev references via fal.ai proxy, runs the blind A/B, and audits the HiDream-I1 model card. Both feed ADR-0006.

### What is already in production (UNTOUCHABLE)

- **FLUX schnell T2I path** on RunPod (Epic 1+2): production endpoint, network volume `mqqgzwnfp1` (used by i2i Story 3.1; do NOT remount), `serverless/handler.py` flux branch, gateway `POST /jobs` route → FLUX, capability discovery advertising image kind.
- **i2i path** (Epic 3, Qwen-Image-Edit fp8) sharing the same handler.
- **Video path** (Epic 5, LTX-Video) on a separate endpoint.
- **Gateway** at `https://gemma4-gateway.jhonata-matias.workers.dev` (Cloudflare Worker).

**Non-negotiable constraint:** This spike must produce **zero diff** in any of the above. Verified at AC8.

### What this story produces (NOT production)

- An **isolated** RunPod L40S serverless endpoint (separate ID from production), running HiDream-I1 Dev FP16 via Diffusers backend, **NEVER advertised by capability discovery, NEVER exposed by gateway routes, NEVER referenced by SDK**.
- A spike directory `spike/hidream-poc/` with handler script, prompt set fixture, measurement runner, and teardown checklist. Parallel pattern to `spike/runs/` already in tree from prior PoCs.
- Raw measurement output written to `.aiox/notes/story-6.1/measurements-{timestamp}.json` (cold-start array, latency array, cost ledger) plus a markdown roll-up at `docs/research/hidream-i1-dev-poc-results.md` (raw section only — Story 6.2 appends evaluation section).
- 50 PNG outputs archived under `.aiox/notes/story-6.1/outputs/` for handoff to Story 6.2.
