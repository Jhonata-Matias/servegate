## User Story

**As** a servegate maintainer evaluating whether HiDream-I1 Dev should replace or complement FLUX schnell as the higher-fidelity T2I path,
**I want** a 50-prompt blind A/B comparison of HiDream PoC outputs (from Story 6.1) against FLUX dev reference images (generated via fal.ai), plus a compliance-grade audit of the HiDream-I1 model card (license, training data, procedência, NSFW behavior), all synthesized into ADR-0006 with adopt / reject / defer verdict + ramp/exit conditions,
**so that** Epic 6 closes with a decision-grade artifact backed by evidence — not speculation — that protects future Epic 7 from late-stage compliance surprises and gives a 6-month revisit baseline if the verdict is reject or defer.

## Context

### Why this story exists now

This is the **synthesis leg** of Epic 6, paired with Story 6.1's measurement leg. While 6.1 produces empirical infrastructure data (cold start, latency, $/imagem) and 50 HiDream-generated PNGs, 6.2 owns:

1. **The evaluation set itself** — 50 prompts category-tagged across servegate's target use cases (photorealistic edges-cases like hands/faces/text-in-image, design assets, mixed/general)
2. **The FLUX dev reference baseline** — generated via fal.ai (legal proxy for non-commercial weights we can't self-host)
3. **The blind A/B comparison** — N=1 single-evaluator (per @po decision; explicit single-evaluator bias caveat baked into ADR)
4. **The model card audit** — license fine print, training data disclosures, HiDream-AI procedência, NSFW filter behavior, output ownership
5. **ADR-0006 itself** — adopt / reject / defer verdict with ramp/exit conditions for downstream Epic 7

The PRD (line 41) locks the success bar **before** the evaluation runs: HiDream wins with **≥ 45% blind A/B win rate vs FLUX dev** → ADR adopts. This pre-commitment mitigates confirmation bias.

### What's already established

- **Story 6.1 (paired):** produces 50 HiDream PNG outputs at standardized config (28 steps, 1024², deterministic seed schedule). Manifest at `.aiox/notes/story-6.1/outputs/manifest.json`.
- **fal.ai as quality reference proxy:** fal.ai hosts FLUX dev under their TOS; we can call it for evaluation references without violating BFL Non-Commercial weight license (we are consumers of fal.ai's hosted service, not redistributors).
- **ADR template pattern:** ADR-0003 (i2i model selection) is the closest precedent — same shape (model selection backed by empirical/research evidence, with cost matrix + rationale + consequences + reversibility sections).

### What this story does NOT cover

- **Infrastructure measurements** (cold start, $/imagem, latency) — Story 6.1's territory; this story consumes 6.1's outputs.
- **Production deployment, SDK changes, gateway routing** — out of scope per Epic 6 PRD (Epic 7's territory if ADR adopts).
- **Recruiting external evaluators** — N=1 per @po decision. ADR notes this as a known limitation.
- **Quantitative comparison to Stable Diffusion 3.5L, Sana, Lumina, etc.** — Epic 6 evaluates HiDream vs FLUX dev as the only A/B pair. ADR can mention alternatives in "rejected paths" section but doesn't run new benchmarks.
