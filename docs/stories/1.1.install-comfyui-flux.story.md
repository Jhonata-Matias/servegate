## Story

**As a** desenvolvedor do projeto gemma4,
**I want** instalar ComfyUI e FLUX.1-schnell no Pod RunPod já em execução, persistido em `/workspace` e auto-iniciando no boot junto com Ollama,
**so that** o stack de inferência ofereça capacidade de geração de imagem via API HTTP (`:8188`) sem perder configuração após `pod.sh stop && pod.sh up` e sem quebrar o Ollama existente.

## Acceptance Criteria

1. ComfyUI clonado e instalado em `/workspace/ComfyUI/` (persistente no network volume)
2. FLUX.1-schnell + dependências (T5 text encoder, CLIP-L, AE/VAE) baixados em `/workspace/ComfyUI/models/{checkpoints,vae,clip,text_encoders}/` com tamanhos de arquivo conferidos (sem downloads parciais)
3. Servidor ComfyUI escutando em `0.0.0.0:8188` após boot, acessível via SSH tunnel
4. `/workspace/start.sh` atualizado para subir ComfyUI **independentemente** do Ollama (falha de um não bloqueia o outro), idempotente (rodar 2x não duplica processos)
5. Smoke test: `POST http://localhost:8188/prompt` com workflow JSON FLUX mínimo (4 steps) retorna `prompt_id` e gera imagem em `/workspace/ComfyUI/output/` em **<30s** após início da geração
6. Coexistência verificada: após ComfyUI subir, `curl http://localhost:11434/api/tags` continua retornando 200 com qwen2.5-coder:14b listado, e `pod.sh status` mostra Pod RUNNING
