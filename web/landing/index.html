01 // Quickstart

Gerando e editando imagens

1

Submeter geração T2I

Submeta um job de text-to-image e aguarde um retorno 202 Accepted.

curl -i -X POST https://gemma4-gateway.jhonata-matias.workers.dev/jobs \
  -H "X-API-Key: $GATEWAY_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "a peaceful zen garden with cherry blossoms, photorealistic",
    "steps": 4,
    "width": 1024,
    "height": 1024,
    "seed": 42
  }'

Retry-After: 5

I2I

Editar uma imagem existente

Use o mesmo POST /jobs. O campo input_image_b64 seleciona Qwen-Image-Edit; o resultado continua chegando por polling em GET /jobs/{job_id}.

  • PNG/JPEG/WebP
    verificado por magic bytes
  • <= 8 MB
    payload decodado
  • <= 1 MP
    downsample defensivo
  • sem 1:1
    use crop não quadrado
INPUT_IMAGE_B64="$(base64 -w 0 input.png)"

curl -i -X POST https://gemma4-gateway.jhonata-matias.workers.dev/jobs \
  -H "X-API-Key: $GATEWAY_API_KEY" \
  -H "Content-Type: application/json" \
  -d "{
    \"prompt\": \"make the jacket green while keeping the background unchanged\",
    \"input_image_b64\": \"$INPUT_IMAGE_B64\",
    \"strength\": 0.85,
    \"steps\": 8,
    \"seed\": 42
  }"

Gerar texto com Gemma 4

Streaming SSE

curl -N -sS https://gemma4-gateway.jhonata-matias.workers.dev/v1/generate \
  -H "Authorization: Bearer $GATEWAY_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "messages": [{"role": "user", "content": "Write one concise haiku about servers."}],
    "max_tokens": 256,
    "temperature": 0.7,
    "stream": true
  }'

Non-Streaming JSON

curl -sS https://gemma4-gateway.jhonata-matias.workers.dev/v1/generate \
  -H "Authorization: Bearer $GATEWAY_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "messages": [{"role": "user", "content": "Say OK"}],
    "max_tokens": 64,
    "stream": false
  }' | jq
02 // Endpoints

API Reference

POST

/jobs

Submit assíncrono para text-to-image ou image-to-image. Sem input_image_b64 usa FLUX.1-schnell; com esse campo usa Qwen-Image-Edit. Esta rota consome o Rate Limit diário e retorna um ID de job para rastreamento.

T2I Request Body

FieldTypeRequired
prompt string Yes (non-empty)
steps integer Yes (rec: 4)
width integer Yes
height integer Yes
seed integer No

I2I Request Body

FieldTypeRequired
prompt string Yes (edit instruction)
input_image_b64 string Yes (PNG/JPEG/WebP)
strength number No (default 0.85)
steps integer No (default 8)
seed integer No

Rate Limiting Header Feedback

  • X-RateLimit-Limit Limite diário Global (Padrão: 100)
  • X-RateLimit-Remaining Quota restante para transações
  • X-RateLimit-Reset Status timestamp para Reset as 00:00 UTC

I2I Validation

  • invalid_image_base64 O campo input_image_b64 não é base64 válido.
  • invalid_aspect_ratio Imagens exatamente 1:1 são rejeitadas. Use crop não quadrado.
  • image_too_large Payload decodado precisa ser <= 8 MB.
  • unsupported_mime_type Apenas PNG, JPEG e WebP por magic bytes.
  • invalid_steps / invalid_strength steps aceita 4-50; strength aceita 0.0 < x <= 1.0.

03 // Troubleshoot

Exceções & SDK

Respostas de Erro

429 rate_limit_exceeded

Quota de uso exaurida, limite diário atingido (100 jobs/dia).

401 invalid_api_key

Falha na verificação de autenticação do Header.

502 bad_gateway · 504 generation_timeout

502: RunPod retornou 5xx ou rede indisponível. 504: RunPod marcou TIMED_OUT (job excedeu COMFY_GENERATION_TIMEOUT_S=280). SDK mapeia 504 para TimeoutError(cause: 'gateway_504').

400 invalid_json 500 config_error

Payload malformado no body, ou erro persistente mascarado do upstream do RunPod.

400 i2i_validation

Chamadas edit() podem retornar invalid_image_base64, image_too_large, unsupported_mime_type, invalid_aspect_ratio, invalid_steps ou invalid_strength.

SDK Client TypeScript

O pacote @jhonata-matias/flux-client@0.3.0 abstrai o polling nos métodos generate() (text-to-image) e edit() (image-to-image via Qwen-Image-Edit self-hosted, Apache 2.0) retornando os seguintes tratamentos nativos:

  • TimeoutError

    Timeout terminal com discriminador cause: poll_exhausted (SDK poll budget), gateway_504 (worker), runpod_timeout (~280s RunPod).

  • RateLimitError ou AuthError

    Erros emitidos na primeira requisição.

  • NetworkError / ValidationError

    Falha de rede ou validação local antes do submit. Para edit(), o SDK valida MIME, 1:1, 8 MB e 1 MP antes da chamada quando possível.
