01 // Quickstart

Gerando a primeira imagem

1

Submeter um request

Submeta um job de geração e aguarde um retorno 202 Accepted.

curl -i -X POST https://gemma4-gateway.jhonata-matias.workers.dev/jobs \
  -H "X-API-Key: $GATEWAY_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "a peaceful zen garden with cherry blossoms, photorealistic",
    "steps": 4,
    "width": 1024,
    "height": 1024,
    "seed": 42
  }'
Expected Response 
HTTP/1.1 202 Accepted
Location: /jobs/2f3f0f1f-2a6f-4d8b-b6f3-0df9d9f36e9e
Retry-After: 5

{
  "job_id": "2f3f0f1f-2a6f-4d8b-b6f3-0df9d9f36e9e",
  "status_url": "/jobs/2f3f0f1f-2a6f-4d8b-b6f3-0df9d9f36e9e",
  "est_wait_seconds": "unknown"
}

Faça Polling do status

02 // Endpoints

API Reference

POST

/jobs

Submit de geração de imagens de forma assíncrona. Esta rota consome o Rate Limit diário e retorna um ID de job para rastreamento.

Request Body

FieldTypeRequired
prompt string Yes (non-empty)
steps integer Yes (rec: 4)
width integer Yes
height integer Yes
seed integer No

Rate Limiting Header Feedback

  • X-RateLimit-Limit Limite diário Global (Padrão: 100)
  • X-RateLimit-Remaining Quota restante para transações
  • X-RateLimit-Reset Status timestamp para Reset as 00:00 UTC
GET

/jobs/{job_id}

Faz pooling do estado de um job submetido. Este endpoint não consome quota ou limite diário.

Mapeamento de Retorno (States)

200 OK
Sucesso
{ output: ... }
202 Accepted
In Progress
Retorna Retry-After: 5
404 Not Found
Expired/Unknown
{ error: "job_not_found_or_expired" }
5xx Errors
Upstream Fail
Timeout ou Runpod Erro Fatal
03 // Troubleshoot

Exceções & SDK

Respostas de Erro

429 rate_limit_exceeded

Quota de uso exaurida, limite diário atigindo (100 jobs/dia).

401 invalid_api_key

Falha na verificação de autenticação do Header.

502 bad_gateway · 504 generation_timeout

502: RunPod retornou 5xx ou rede indisponível. 504: RunPod marcou TIMED_OUT (job excedeu COMFY_GENERATION_TIMEOUT_S=280). SDK mapeia 504 para TimeoutError(cause: 'gateway_504').

400 invalid_json 500 config_error

Payload mal formato no Body, ou erro persistente mascara do upstream do RunPod.

SDK Client TypeScript

O pacote @jhonata-matias/flux-client@0.2.0 abstrai de forma elegante toda essa camada de pooling no método generate() retornando os seguintes tratamentos nativos:

  • TimeoutError

    Timeout terminal com discriminador cause: poll_exhausted (SDK poll budget), gateway_504 (worker), runpod_timeout (~280s RunPod).

  • RateLimitError ou AuthError

    Erros emitidos na primeira requisição.

  • NetworkError / ValidationError

    Falha ao iniciar payload na base local.
