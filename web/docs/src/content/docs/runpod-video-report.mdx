## Executive verdict

The evidence supports **RunPod Serverless Flex + LTX-Video 2B + L4/A5000/RTX 4090 24GB-class GPUs** as the alpha lane for both text-to-video and image-to-video. T2V validation closed in April 2026; I2V validation closed on 2026-05-04 after the production-readiness gate forced re-running smokes inside the committed envelope (Story 5.1.1.3 had run on L40S 48GB, outside the envelope, so its evidence was insufficient).

<Aside type="tip" title="Decision boundary">
Treat “fits on GPU” and “provisions reliably” as separate questions. LTX-Video 2B fits the target GPU class with ~5GB of VRAM headroom. RunPod provisioning is healthy when workers are warm; first-call cold-pool spawn (~10 min wallclock observed once) remains the watch item.
</Aside>

## Cost envelope

RunPod prices the L4/A5000/RTX 4090 24GB Serverless Flex tier at **US$0.00019–0.00029/s** depending on GPU type. Empirical re-validation on NVIDIA RTX A5000 produced **86.6 GPU-seconds median** (down from the 198.05s assumed pre-gate; LTX 2B distilled is materially faster than the original report assumption).

These numbers are 40% lower than the original v1.4 estimate due to faster execution on the actual 24GB-class GPU. They exclude R2 storage/egress and exclude any decision to keep `workersMin=1` warm. If `workersMin=1` is enabled, the cost profile changes from burst compute to continuous capacity (~US$5–15/month idle).

## Recommended deployment posture

1. Keep RunPod for **alpha T2V and I2V on LTX 2B distilled**.
2. I2V re-validated in the 24GB-class envelope on 2026-05-04 (3/3 GREEN on NVIDIA RTX A5000). The gateway video route and SDK are unblocked; first-run cold-start (~10 min wallclock when pool spawns) must be surfaced via async polling UX.
3. Avoid `networkVolumeId` by default; use `MODEL_LOADING_PATH=pretrained` until the product has enough traffic to justify a volume cache.
4. Keep the endpoint async-only. Do not use synchronous gateway calls for video.
5. Escalate to hosted API fallback if repeated `IN_QUEUE` windows exceed the agreed operational threshold.
