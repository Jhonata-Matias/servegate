## Executive verdict

The evidence supports **RunPod Serverless Flex + LTX-Video 2B + L4/A5000/3090 24GB-class GPUs** as the first alpha lane for text-to-video. The same evidence does **not** support a production-ready claim for all video modes, because I2V is still missing a successful fixed-handler smoke run and recent jobs remained queued without worker allocation.

<Aside type="caution" title="Decision boundary">
Treat “fits on GPU” and “provisions reliably” as separate questions. LTX-Video 2B fits the target GPU class. RunPod provisioning still needs more evidence before beta.
</Aside>

## Cost envelope

RunPod currently prices the L4/A5000/3090 Serverless Flex class at **US$0.00019/s**. Using the local median of **198.05 GPU-seconds** per cold call:

<section class="cost-panel" aria-label="RunPod cost estimate">
	<div>
		<span>Per cold call</span>
		<strong>~US$0.0376</strong>
	</div>
	<div>
		<span>20 calls/day</span>
		<strong>~US$22.60/month</strong>
	</div>
	<div>
		<span>100 calls/day</span>
		<strong>~US$112.90/month</strong>
	</div>
</section>

These numbers exclude R2 storage/egress and any decision to keep active workers warm. If `workersMin=1` is enabled, the cost profile changes from burst compute to continuous capacity.

## Recommended deployment posture

1. Keep RunPod for **alpha T2V LTX 2B**.
2. Keep I2V behind capability discovery as **validation pending** until Story 5.1.1.2 closes GREEN.
3. Avoid `networkVolumeId` by default; use `MODEL_LOADING_PATH=pretrained` until the product has enough traffic to justify a volume cache.
4. Keep the endpoint async-only. Do not use synchronous gateway calls for video.
5. Escalate to hosted API fallback if repeated `IN_QUEUE` windows exceed the agreed operational threshold.
